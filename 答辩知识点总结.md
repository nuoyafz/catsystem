### 1. 统一返回结果封装

**知识点**：前后端分离架构中，所有接口返回统一格式，前端统一处理。

**核心代码**：`Result.java`

```java
@Data
public class Result<T> {
    private Integer code;
    private String message;
    private T data;

    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("操作成功");
        result.setData(data);
        return result;
    }

    public static <T> Result<T> error(String message) {
        Result<T> result = new Result<>();
        result.setCode(500);
        result.setMessage(message);
        return result;
    }
}
```

**老师可能问**：
- **为什么用泛型 `<T>`？** → 让 data 字段可以返回任意类型（User、List、Page 等），避免每个接口写一个返回类。
- **为什么 code 用 Integer？** → HTTP 状态码是整数，200 表示成功，500 表示失败，前端根据 code 判断。

### 2. JWT 身份认证

**知识点**：JWT（JSON Web Token）是无状态认证方案，由三部分组成：Header（算法）、Payload（数据）、Signature（签名）。

**核心代码**：`JwtUtils.java`

```java
@Component
public class JwtUtils {

    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expiration}")
    private Long expiration;

    // 生成 Token
    public String generateToken(Long userId, String username, String role) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("userId", userId);
        claims.put("username", username);
        claims.put("role", role);

        return Jwts.builder()
                .setClaims(claims)
                .setIssuedAt(new Date())
                .setExpiration(new Date(System.currentTimeMillis() + expiration))
                .signWith(SignatureAlgorithm.HS512, secret)
                .compact();
    }

    // 解析 Token
    public Claims parseToken(String token) {
        return Jwts.parser()
                .setSigningKey(secret)
                .parseClaimsJws(token)
                .getBody();
    }

    // 验证 Token
    public boolean validateToken(String token) {
        try {
            Claims claims = parseToken(token);
            return !claims.getExpiration().before(new Date());
        } catch (Exception e) {
            return false;
        }
    }
}
```

**老师可能问**：
- **JWT 和 Session 的区别？** → Session 存储在服务器，JWT 存储在客户端；JWT 更适合分布式系统。
- **HS512 是什么？** → HMAC-SHA512 签名算法，用 secret 密钥对 Header+Payload 签名，防止篡改。
- **Token 过期了怎么办？** → 前端捕获 401 错误，跳转登录页重新获取 Token。
- **JWT 的 payload 可以被看到吗？** → 可以，payload 只是 Base64 编码，不是加密。敏感信息不要放 payload。

### 3. 跨域配置（CORS）

**知识点**：前端（localhost:5173）和后端（localhost:8080）端口不同，浏览器同源策略会拦截请求，需要配置 CORS。

**核心代码**：`CorsConfig.java`

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOriginPatterns("*")
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("*")
                .allowCredentials(true)
                .maxAge(3600);
    }
}
```

**老师可能问**：
- **什么是同源策略？** → 协议 + 域名 + 端口三者都相同才算同源。
- **allowCredentials(true) 的作用？** → 允许携带 Cookie / Authorization 头。
- **maxAge(3600) 是什么？** → 预检请求（OPTIONS）的缓存时间，3600 秒内不再发预检请求。

### 4. 全局异常处理

**知识点**：使用 `@RestControllerAdvice` 统一捕获所有 Controller 抛出的异常，避免每个接口都写 try-catch。

**核心代码**：`GlobalExceptionHandler.java`

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        e.printStackTrace();
        return Result.error("服务器内部错误：" + e.getMessage());
    }

    @ExceptionHandler(RuntimeException.class)
    public Result<Void> handleRuntimeException(RuntimeException e) {
        e.printStackTrace();
        return Result.error("运行时错误：" + e.getMessage());
    }
}
```

**老师可能问**：
- **@RestControllerAdvice 和 @ControllerAdvice 的区别？** → `@RestControllerAdvice` = `@ControllerAdvice` + `@ResponseBody`，直接返回 JSON。
- **@ExceptionHandler 的匹配顺序？** → 优先匹配最具体的异常类型（RuntimeException 比 Exception 更具体）。

### 5. MyBatis-Plus 分页配置

**知识点**：MyBatis-Plus 分页需要注册 PaginationInnerInterceptor 插件，否则分页查询不会生效。

**核心代码**：`MyBatisPlusConfig.java`

```java
@Configuration
public class MyBatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        return interceptor;
    }

    @Bean
    public MetaObjectHandler metaObjectHandler() {
        return new MetaObjectHandler() {
            @Override
            public void insertFill(MetaObject metaObject) {
                this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, LocalDateTime.now());
                this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
            }

            @Override
            public void updateFill(MetaObject metaObject) {
                this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
            }
        };
    }
}
```

**老师可能问**：
- **分页插件的原理？** → 拦截 SQL 语句，自动添加 `LIMIT` 子句，同时执行 `COUNT(*)` 查询总数。
- **MetaObjectHandler 的作用？** → 自动填充 `createTime`、`updateTime` 字段，不需要手动 set。

### 6. 用户登录与注册

**知识点**：登录验证密码、生成 Token；注册加密密码、设置默认角色。

**核心代码**：`UserController.java`

```java
// 登录
@PostMapping("/login")
public Result<Map<String, Object>> login(@RequestBody User user) {
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(User::getUsername, user.getUsername());
    User dbUser = userService.getOne(wrapper);

    if (dbUser == null) {
        return Result.error("用户名不存在");
    }
    if (!PasswordUtils.verify(user.getPassword(), dbUser.getPassword())) {
        return Result.error("密码错误");
    }
    if (dbUser.getStatus() == 0) {
        return Result.error("账号已被禁用");
    }

    String token = jwtUtils.generateToken(dbUser.getId(), dbUser.getUsername(), dbUser.getRole());
    Map<String, Object> data = new HashMap<>();
    data.put("token", token);
    data.put("user", dbUser);
    return Result.success(data);
}

// 注册
@PostMapping("/register")
public Result<Void> register(@RequestBody User user) {
    LambdaQueryWrapper<User> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(User::getUsername, user.getUsername());
    if (userService.getOne(wrapper) != null) {
        return Result.error("用户名已存在");
    }
    user.setPassword(PasswordUtils.encrypt(user.getPassword()));
    user.setRole("user");
    user.setStatus(1);
    userService.save(user);
    return Result.success();
}
```

**密码加密**：`PasswordUtils.java`

```java
public class PasswordUtils {
    public static String encrypt(String password) {
        return DigestUtil.md5Hex(password);  // Hutool MD5
    }
    public static boolean verify(String rawPassword, String encryptedPassword) {
        return encrypt(rawPassword).equals(encryptedPassword);
    }
}
```

**老师可能问**：
- **为什么用 LambdaQueryWrapper？** → 类型安全，编译期检查，避免手写 SQL 拼写错误。
- **MD5 安全吗？** → MD5 不够安全（可彩虹表攻击），生产环境应加盐（Salt）或用 BCrypt。课程项目中用于演示。

### 1. 分页查询 + 条件筛选

**知识点**：MyBatis-Plus 的 `Page` 对象 + `LambdaQueryWrapper` 实现分页和条件查询。

**核心代码**：`CatController.java`

```java
@GetMapping("/page")
public Result<Page<Cat>> getPage(
        @RequestParam(defaultValue = "1") Integer pageNum,
        @RequestParam(defaultValue = "10") Integer pageSize,
        @RequestParam(required = false) String name,
        @RequestParam(required = false) String healthStatus) {

    Page<Cat> page = new Page<>(pageNum, pageSize);
    LambdaQueryWrapper<Cat> wrapper = new LambdaQueryWrapper<>();

    if (StringUtils.hasText(name)) {
        wrapper.like(Cat::getName, name);  // 模糊查询
    }
    if (StringUtils.hasText(healthStatus)) {
        wrapper.eq(Cat::getHealthStatus, healthStatus);  // 精确匹配
    }
    wrapper.orderByDesc(Cat::getCreateTime);

    return Result.success(catService.page(page, wrapper));
}
```

**老师可能问**：
- **like 和 eq 的区别？** → `like` 是模糊匹配（SQL `LIKE '%xxx%'`），`eq` 是精确匹配（SQL `=`）。
- **为什么用 `StringUtils.hasText`？** → 同时判断 null、空字符串、纯空格，比 `!= null` 更安全。
- **Page 对象返回什么？** → 包含 `records`（数据列表）、`total`（总数）、`pages`（总页数）、`current`（当前页）。

### 2. 多表关联查询（自定义 SQL）

**知识点**：MyBatis-Plus 的 `BaseMapper` 只能做单表操作，多表关联需要自定义 SQL。

**核心代码**：`FeedRecordMapper.java`

```java
@Mapper
public interface FeedRecordMapper extends BaseMapper<FeedRecord> {

    @Select("SELECT r.*, c.name as cat_name, u.username, " +
            "COALESCE(v.real_name, u.username) as real_name " +
            "FROM t_feed_record r " +
            "LEFT JOIN t_cat c ON r.cat_id = c.id " +
            "LEFT JOIN t_user u ON r.user_id = u.id " +
            "LEFT JOIN t_volunteer v ON r.user_id = v.user_id " +
            "ORDER BY r.feed_time DESC")
    List<FeedRecord> selectAllWithDetails();
}
```

**重要规则**：自定义 SQL 方法必须直接注入 Mapper 接口，不能通过 `service.getBaseMapper()` 调用。

```java
@Autowired
private FeedRecordMapper feedRecordMapper;  // 正确

// feedRecordMapper.selectAllWithDetails();  // 正确调用
```

**实体类非数据库字段**：`FeedRecord.java`

```java
@TableField(exist = false)
private String catName;      // 关联查询的猫咪名称
@TableField(exist = false)
private String username;     // 关联查询的用户名
@TableField(exist = false)
private String realName;     // 关联查询的真实姓名
```

**老师可能问**：
- **为什么用 LEFT JOIN 而不是 INNER JOIN？** → LEFT JOIN 保留投喂记录，即使猫咪/用户被删除了记录还在。
- **COALESCE 是什么？** → SQL 函数，如果第一个参数为 NULL 则返回第二个参数。这里优先显示志愿者真实姓名，没有则显示用户名。
- **@TableField(exist = false) 的作用？** → 告诉 MyBatis-Plus 这个字段不是数据库列，插入/更新时忽略。

**老师可能问**：
- **`() => import(...)` 是什么？** → 路由懒加载，只有访问该路由时才加载对应组件，减少首屏加载时间。
- **createWebHistory 和 createHashHistory 的区别？** → WebHistory 使用 HTML5 History API（URL 没有 #），HashHistory 使用 URL 哈希（`/#/volunteer`）。

### 5. Service 层实现

**知识点**：MyBatis-Plus 的 `ServiceImpl` 已经内置了 CRUD 方法，简单模块不需要写额外逻辑。

**核心代码**：`ActivityServiceImpl.java`

```java
@Service
public class ActivityServiceImpl extends ServiceImpl<ActivityMapper, Activity>
        implements ActivityService {
    // 继承 ServiceImpl 后自动拥有 save/update/remove/list 等方法
}
```

**老师可能问**：
- **ServiceImpl 提供了哪些方法？** → `save()`、`saveBatch()`、`updateById()`、`removeById()`、`list()`、`getById()`、`page()` 等。
- **什么时候需要自定义 Service 方法？** → 需要多表关联、复杂业务逻辑、事务控制时。

| 知识点 | 一句话回答 |
|--------|-----------|
| BigDecimal vs Double | BigDecimal 精确计算，Double 有精度丢失风险 |

| LEFT JOIN 关联查询 | 志愿者表关联用户表获取 username |

## 四、通用答辩知识点（三人都可能问到）

### 1. 项目架构

```
前端 (Vue 3 + Element Plus)
    ↓  HTTP/JSON (Axios)
后端 (Spring Boot + MyBatis-Plus)
    ↓  JDBC
数据库 (MySQL 8.0)
```

**老师可能问**：
- **为什么选 Spring Boot？** → 开箱即用、自动配置、内嵌 Tomcat、生态完善。
- **为什么选 Vue 3？** → Composition API 逻辑复用更好、性能提升、TypeScript 友好。

### 2. 数据库设计

7 张核心表：
- `t_user` — 用户表
- `t_cat` — 流浪猫表
- `t_feed_record` — 投喂记录表
- `t_volunteer` — 志愿者表
- `t_activity` — 活动表
- `t_cat_tag` — 标签表
- `t_cat_tag_relation` — 猫-标签关联表

**老师可能问**：
- **数据库设计遵循什么范式？** → 至少第三范式（3NF），消除传递依赖，避免数据冗余。
- **外键为什么不在数据库层面设置？** → 代码层面维护关联，提高性能，避免级联删除的副作用。

### 3. 安全性

| 安全措施 | 实现方式 |
|---------|---------|
| 密码加密 | MD5（Hutool DigestUtil） |
| 身份认证 | JWT Token |
| 跨域控制 | CorsConfig 白名单 |
| SQL 注入防护 | MyBatis-Plus 参数化查询 |
| 异常处理 | GlobalExceptionHandler 统一捕获 |
